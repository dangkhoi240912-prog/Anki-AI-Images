1. MULTI-KEY GEMINI ARCHITECTURE (v4.2)
   ──────────────────────────────────────
   
   Keyword Generation:
   Key #1 (primary) → Key #4 (backup) ✨ NEW
   
   Image Evaluation:
   Key #2 (primary) → Key #2-Backup → Key #3 → Key #1 ✨ NEW levels
   
   Status: Thread-safe, automatic fallback, logged


2. RATE LIMIT PROTECTION (v4.2)
   ──────────────────────────────
   
   ┌─────────────────────┐
   │  HTTP Response      │
   └─────────────────────┘
            ↓
   Status = 429/503/403?
            ↓ YES
   ┌─────────────────────────────────┐
   │ RateLimitHandler.mark_paused()  │
   │ Provider marked for 60 seconds  │
   └─────────────────────────────────┘
            ↓
   ┌─────────────────────────┐
   │ Skip to next provider   │
   │ Fail silently           │
   │ Log for debugging       │
   └─────────────────────────┘
            ↓ After 60s
   ┌─────────────────────────┐
   │ Provider re-enabled     │
   │ Automatically          │
   └─────────────────────────┘

┌─────────────────────────────────────────────────────────────────────┐
│                    BACKWARD COMPATIBILITY                           │
└─────────────────────────────────────────────────────────────────────┘

Old Config (v4.0):
  - 3 Gemini keys
  - 7 image providers
  - No rate limit protection

New Config (v4.2):
  - 4 Gemini keys (with 2 backups) ✨ +2
  - 15+ image providers ✨ +8
  - Rate limit protection ✨ NEW
  - Configurable pause duration ✨ NEW

Flow if old config detected:
  1. Load existing keys (backwards compatible)
  2. New keys default to empty string
  3. New settings default to sensible values
  4. User can gradually upgrade settings
  5. No forced migration, no data loss
